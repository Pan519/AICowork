## 1. 产品介绍

### 1.1 什么是 AICowork？

AICowork 是一款基于 Claude AI 的智能对话助手，具有以下特点：

- 🤖 **智能对话** - 使用最先进的 Claude AI 模型
- 🧠 **记忆系统** - 本地持久化知识库，支持智能搜索
- 🔌 **可扩展** - 支持 MCP 服务器、插件、技能等扩展
- ⚙️ **高度可配置** - 丰富的设置选项满足不同需求
- 🔒 **隐私安全** - 数据本地存储，保护用户隐私
- 🌍 **多语言** - 支持中文和英文界面

### 1.2 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 (64位) |
| 内存 | 4GB 及以上（推荐 8GB） |
| 磁盘空间 | 500MB 可用空间 |
| 网络 | 需要联网访问 Claude API |
| 其他 | 需要有效的 Claude API Key |

### 1.3 应用场景

- 💻 **编程开发** - 代码编写、调试、解释
- 📝 **文档写作** - 撰写、翻译、润色文档
- 📚 **学习辅助** - 解答疑问、总结要点
- 🎨 **创意工作** - 头脑风暴、创意生成
- 📊 **数据分析** - 数据解读、报告生成
- 🤝 **协作交流** - 团队沟通、会议记录

## 2. 安装与卸载

### 2.1 安装步骤

#### 方式一：安装包安装

```
1. 下载 AICowork-x.x.x.exe 安装文件
2. 双击运行安装程序
3. 选择安装路径（默认：C:\Users\用户名\AppData\Local\Programs\aicowork）
4. 点击"安装"按钮
5. 等待安装完成
6. 点击"完成"启动应用
```

#### 方式二：绿色版（免安装）

```
1. 下载 AICowork-x.x.x-win.zip
2. 解压到任意目录
3. 双击 AICowork.exe 启动
```

### 2.2 首次启动

首次启动时会：

1. 创建配置目录：`%USERPROFILE%\.aicowork`
2. 创建记忆数据库：`%USERPROFILE%\.aicowork\memvid\memory.mv2`
3. 创建日志目录：`%USERPROFILE%\.aicowork\logs\`
4. 显示欢迎界面

### 2.3 卸载

#### Windows 程序卸载

```
1. 打开"设置" → "应用"
2. 找到"AICowork"
3. 点击"卸载"
4. 确认卸载
```

#### 手动清理（可选）

卸载后如需完全清理：

```
删除以下目录和文件：
- %USERPROFILE%\.aicowork\
- %APPDATA%\aicowork\
```

## 5. AI 对话功能

### 5.1 基础对话

#### 发送消息

```
方式1: 在输入框输入文字 → 按 Enter
方式2: 在输入框输入文字 → 点击"发送"按钮
```

#### 换行输入

```
按 Shift + Enter 可以换行（不发送消息）
```

#### 新建对话

```
点击侧边栏"+ 新对话"按钮
或按快捷键 Ctrl + N
```

#### 切换对话

```
在侧边栏点击任意历史对话即可切换
```

#### 删除对话

```
鼠标悬停在对话卡片上
点击删除图标（🗑️）
```

### 5.2 对话类型

#### 普通对话

- 默认的对话模式
- 保持上下文连续性
- 支持多轮对话

#### 会话对话（带工作目录）

```
创建时选择工作目录
AI 可以访问指定目录的文件
适合项目相关的对话
```

### 5.3 模型配置

#### 支持的厂商

| 厂商 | 说明 |
|------|------|
| **Anthropic** | Claude 官方 API |
| **Azure** | Azure OpenAI Service |
| **其他** | 兼容 Claude API 的服务 |

#### 模型参数

| 参数 | 说明 | 范围 | 默认值 |
|------|------|------|--------|
| **Temperature** | 创造性程度 | 0.0-2.0 | 0.7 |
| **Max Tokens** | 最大回复长度 | 1-200K | 8192 |
| **Top P** | 核采样 | 0.0-1.0 | - |

### 5.4 多配置管理

#### 创建新配置

```
1. 设置 → Claude API
2. 点击"查看列表"查看所有配置

4. 填写新的配置信息
5. 保存
```

#### 切换配置

```
1. 在配置列表中找到要使用的配置

```

#### 编辑/删除配置

```
编辑：点击配置卡片上的编辑按钮
```

## 6. 记忆系统

### 6.1 什么是记忆系统？

记忆系统是本地化的持久化知识库，功能包括：

- 📝 存储任意文本信息
- 🔍 智能语义搜索
- 🤖 RAG（检索增强生成）问答
- 🏷️ 标签分类管理
- 💾 导入/导出功能

### 6.2 配置记忆系统

#### 基本设置

```
设置 → 记忆系统

选项：
✅ 启用记忆功能 - 开启/关闭整个记忆系统
✅ 自动存储记忆 - 会话结束时自动保存重要内容
搜索模式: 词法/语义/自动
默认搜索结果数: 3-20
```

#### 搜索模式说明

| 模式 | 原理 | 优点 | 适用场景 |
|------|------|------|----------|
| **词法搜索** | BM25 关键词匹配 | 速度快，精确匹配 | 已知关键词搜索 |
| **语义搜索** | 向量相似度匹配 | 理解语义，相关性高 | 概念性搜索 |
| **自动混合** | 智能选择最佳方式 | 综合性能最优 | 日常使用推荐 |

### 6.3 标签管理

#### 创建标签

```
1. 设置 → 记忆系统 → 标签管理
2. 点击"+ 新建标签"
3. 输入标签名称
4. 添加
```

#### 使用标签

```
- 添加记忆时选择标签
- 搜索时按标签筛选
- AI 可以自动给记忆打标签
```

### 6.4 添加记忆

#### 方式一：手动添加

```
1. 设置 → 记忆系统
2. 点击"+ 添加新记忆"
3. 填写信息：
   - 标题：简短描述
   - 内容：详细信息
   - 分类：选择分类
   - 标签：选择标签
4. 保存
```

#### 方式二：斜杠命令

```
/memory save
```

#### 方式三：自动存储

```
1. 启用"自动存储记忆"
2. 选择自动存储的分类
3. 对话结束时自动保存
```

### 6.5 记忆分类

| 分类 | 图标 | 用途 | 示例 |
|------|------|------|------|
| **项目相关** | 📁 | 项目信息、配置、结构 | 项目根目录、依赖列表 |
| **用户偏好** | ⚙️ | 个人习惯、设置偏好 | 喜欢的代码风格、常用参数 |
| **技术记录** | 💡 | 技术要点、代码片段 | 设计模式、算法实现 |
| **上下文信息** | 📝 | 背景、上下文 | 项目背景、业务逻辑 |
| **对话摘要** | 💬 | 重要对话总结 | 决策记录、讨论结果 |
| **自定义** | 🏷️ | 用户自定义 | 任意自定义内容 |

### 6.6 搜索记忆

#### 基础搜索

```
1. 在搜索框输入关键词
2. 按 Enter 或点击搜索按钮
3. 查看结果和相关度分数
```

#### 高级搜索技巧

```
- 使用多个关键词提高准确度
- 尝试同义词
- 切换搜索模式
- 增加搜索结果数量
```

### 6.7 AI 问答（RAG）

基于记忆的智能问答：

```
1. 点击"💬 AI 问答"按钮
2. 输入问题
3. AI 会：
   - 搜索相关记忆
   - 基于记忆内容回答
   - 引用来源
```

### 6.8 编辑和删除

#### 编辑记忆

```
1. 找到要编辑的记忆卡片
2. 点击编辑按钮
3. 修改内容
4. 保存
```

#### 删除记忆

```
单条删除：
- 点击记忆卡片的删除按钮

批量删除：
1. 点击"多选"按钮
2. 勾选要删除的记忆
3. 点击"删除选中"
```

### 6.9 导入/导出

#### 导出记忆

```
1. 进入多选模式
2. 勾选要导出的记忆
3. 点击"导出选中"
4. 保存为 JSON 文件
```

#### 导入记忆

```
1. 点击"导入记忆"按钮
2. 选择 JSON 文件
3. 确认导入
```

## 7. 技能系统

### 7.1 什么是技能？

技能（Skills）是预定义的 AI 功能集合，可以：

- 🎯 完成特定任务
- 🔄 复用常用提示词
- 🚀 提高工作效率
- 🎨 自定义 AI 行为

### 7.2 技能类型

#### 内置技能

系统自带的技能：

- `/commit` - Git 提交消息生成
- `/review-pr` - PR 代码审查
- `/pdf` - PDF 文档分析
- 等等...

#### 自定义技能

用户创建的技能：

```
设置 → 技能管理 → + 新建技能
```

### 7.3 创建技能

#### 基本步骤

```
1. 设置 → 技能管理
2. 点击"+ 新建技能"
3. 填写信息：
   - 技能名称：如"代码审查"
   - 技能描述：简要说明功能
   - 提示词模板：AI 执行的指令
   - 参数：可变参数
4. 保存
```

#### 提示词模板示例

```
你是一位专业的代码审查员。请审查以下代码：
重点关注：
- 代码风格
- 潜在 bug
- 性能问题
- 安全问题

代码：
{code}

请给出详细的审查意见。
```

### 7.4 使用技能

#### 方式一：斜杠命令

```
/技能名称 参数
```

#### 方式二：在对话中调用

```
在对话中直接输入技能名称
```

### 7.5 管理技能

```
1. 设置 → 技能管理
2. 找到要编辑的技能
3. 点击编辑按钮
4. 修改内容
5. 保存
```

#### 删除技能

```
1. 找到要删除的技能
2. 点击删除按钮
3. 确认删除
```

#### 导入/导出技能

```
导出：
1. 选择要导出的技能
2. 点击"导出"
3. 保存为 JSON

导入：
1. 点击"导入技能"
2. 选择 JSON 文件
3. 确认导入
```

### 7.6 技能标签

#### 创建标签

```
1. 设置 → 技能管理 → 标签管理
2. 点击"+ 新建标签"
3. 输入标签名称
```

#### 使用标签

```
- 给技能打标签
```

## 8. MCP 服务器

MCP（Model Context Protocol）是一个扩展协议，允许 AI 访问外部工具和服务。

### 8.2 启用 MCP

```
1. 设置 → MCP 服务器

3. 配置服务器列表
```

### 8.3 添加 MCP 服务器

```
1. 点击"+ 添加服务器"
2. 填写信息：
   - 服务器名称
   - 服务器地址
   - 认证信息（如需要）
3. 保存
```

### 8.4 使用 MCP 工具

配置后，AI 可以使用 MCP 提供的工具：

```
用户：帮我查询天气
AI：[使用 MCP 天气工具] 今天北京晴天...
```

## 11. 钩子系统

### 11.1 什么是钩子？

钩子（Hooks）允许在特定事件发生时执行自定义操作。

### 11.2 钩子类型

| 事件 | 说明 |
|------|------|
| **beforeSend** | 发送消息前 |
| **afterReceive** | 接收回复后 |
| **onError** | 发生错误时 |
| **onSessionStart** | 会话开始时 |
| **onSessionEnd** | 会话结束时 |

### 11.3 创建钩子

```
1. 设置 → 钩子
2. 点击"+ 新建钩子"
3. 选择事件类型
4. 编写钩子脚本
5. 保存
```

## 16. 常见问题

### 16.1 安装和启动

**Q: 安装后无法启动？**

```
A: 检查以下几点：
1. 确认系统版本（需要 Windows 10/11）
2. 检查是否有杀毒软件拦截
3. 以管理员身份运行
4. 查看日志文件了解详细错误
```

**Q: 应用启动很慢？**

```
A: 首次启动需要加载记忆索引，可能需要几秒钟
   后续启动会更快
```

### 16.2 API 相关

**Q: 如何获取 API Key？**

详见 [3.2 获取 API Key](#32-获取-api-key)

**Q: 提示"API Key 未配置"？**

```
A: 设置 → Claude API → 粘贴 API Key → 保存
```

**Q: API 调用失败？**

```
A: 检查：
1. API Key 是否正确
2. 网络连接是否正常
3. API 服务是否可用
4. 账户余额是否充足
```

### 16.3 记忆系统

**Q: 记忆搜索没有结果？**

```
A: 尝试：
1. 确认已添加记忆内容
2. 使用不同的关键词
3. 切换搜索模式
4. 增加搜索结果数量
```

**Q: 如何备份记忆数据？**

详见 [6.9 导入/导出](#69-导入导出)

**Q: 记忆数据存储在哪里？**

```
C:\Users\你的用户名\.aicowork\memvid\memory.mv2
```

### 16.4 性能问题

**Q: 应用占用内存过高？**

```
A: AICowork 内存占用通常在 200-500MB
   如果过高：
1. 重启应用
2. 清空历史对话
3. 清理记忆数据
```

### 16.5 其他问题

**Q: 如何重置所有设置？**

```
A: 删除配置文件：
   C:\Users\你的用户名\.aicowork\config.json
   重启应用将恢复默认设置
```

**Q: 数据会上传到云端吗？**

```
A: 除了调用 Claude API 的对话内容，
   所有记忆、设置等数据都存储在本地
```
